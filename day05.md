Based on the DevOps mock interview recording, here is the complete list of scenario-based questions asked, covering Git and Jenkins:

### **Git Interview Questions**

1.  **Sensitive Data Exposure:** You have accidentally committed sensitive information (e.g., API keys or passwords) to your Git repository. How would you handle this situation? [[15:01](http://www.youtube.com/watch?v=FK__FyfHN8Q&t=901)]
2.  **Incorporate Changes:** You are working on a feature branch and need to incorporate changes from the Main branch into your feature branch. How would you do this? [[15:59](http://www.youtube.com/watch?v=FK__FyfHN8Q&t=959)]
3.  **Fixing a Bug in History:** You have made several commits on your feature branch but realized that one of the earlier commits introduced a bug. How would you fix this without losing the subsequent changes on your branch? [[16:57](http://www.youtube.com/watch?v=FK__FyfHN8Q&t=1017)]
4.  **Team Collaboration:** You have been working on a feature branch for some time and want to share your progress with your team for review. What is the standard workflow to do this? [[18:47](http://www.youtube.com/watch?v=FK__FyfHN8Q&t=1127)]
5.  **Accidental Deletion:** You have accidentally deleted a file from your local repository that you still need. How would you recover it? [[19:37](http://www.youtube.com/watch?v=FK__FyfHN8Q&t=1177)]

### **Jenkins Interview Questions**

6.  **Automated Triggers:** How would you set up a Jenkins job to automatically trigger a build whenever changes are pushed to your Git repository? [[20:16](http://www.youtube.com/watch?v=FK__FyfHN8Q&t=1216)]
7.  **Pipeline Integration:** You have a Jenkins pipeline job that deploys an application to a testing environment. How would you integrate automated testing into this existing pipeline? [[21:03](http://www.youtube.com/watch?v=FK__FyfHN8Q&t=1263)]
8.  **Troubleshooting Failures:** You are experiencing failures in your Jenkins builds. How would you investigate and troubleshoot these failures? [[21:40](http://www.youtube.com/watch?v=FK__FyfHN8Q&t=1300)]
9.  **Disk Space Management:** Your Jenkins server is running low on disk space. How would you address this issue to ensure builds continue to run? [[22:10](http://www.youtube.com/watch?v=FK__FyfHN8Q&t=1330)]
10. **Parameterization:** You need to parameterize a Jenkins job to accept user input during execution (e.g., choosing a branch or environment). How would you implement this? [[22:40](http://www.youtube.com/watch?v=FK__FyfHN8Q&t=1360)]
